# 小冋记事 备忘录系统设计

## 概述
为 小冋记事 应用添加一个平行于现有待办事项功能的备忘录系统。备忘录作为纯笔记功能，与待办事项共享相同的应用框架但具有独立的数据模型和用户界面。

## 设计目标
1. **非侵入式集成**：不影响现有待办事项功能
2. **功能完整性**：满足备忘录核心需求（创建、编辑、搜索、组织）
3. **用户体验一致性**：复用现有设计语言和交互模式
4. **技术可扩展性**：为未来功能扩展奠定基础

## 用户需求
- 备忘录作为纯笔记，无优先级、截止日期或完成状态
- 支持标题、内容、标签、颜色标记、创建/修改时间、附件
- 顶部标签切换在待办事项和备忘录之间导航
- 支持标签筛选、搜索功能、时间排序
- 操作功能：编辑、删除、复制、归档、导出
- 使用 IndexedDB 存储数据

## 技术架构

### 组件结构
```
src/
├── components/
│   ├── memo/
│   │   ├── MemoList.jsx         # 备忘录列表容器
│   │   ├── MemoCard.jsx         # 单个备忘录卡片
│   │   ├── MemoForm.jsx         # 创建/编辑表单
│   │   └── MemoFilters.jsx      # 筛选侧边栏
│   └── shared/
│       ├── Modal.jsx            # 通用模态框
│       └── ColorPicker.jsx      # 颜色选择器
├── hooks/
│   └── useMemo.js               # 备忘录数据 Hook
└── services/
    └── memoService.js           # 数据服务
```

### 数据模型
```javascript
Memo = {
  id: 'string',           // 唯一标识
  title: 'string',        // 备忘录标题
  content: 'string',      // 详细内容
  tags: ['work', 'personal', 'ideas'], // 标签数组
  color: 'blue',          // 颜色标记：blue/green/red/yellow/purple
  isArchived: false,      // 是否已归档
  createdAt: 'timestamp', // 创建时间
  updatedAt: 'timestamp', // 最后修改时间
  attachments: []         // 附件数组
}
```

### 存储方案
- **待办事项**：继续使用 localStorage (`taskflow_tasks`)
- **备忘录**：使用 IndexedDB (`taskflow_db`)，支持高级查询和扩展
- **数据隔离**：两种功能数据完全独立，互不影响
